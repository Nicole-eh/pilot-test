## 🚀 快速开始

### 启动服务器

```bash
# 方式 1：直接运行
node server.js

# 方式 2：使用 npm（如果在 package.json 中配置了）
npm run server
```

服务器将在 `http://localhost:3000` 启动。

### 访问主页

打开浏览器访问：
```
http://localhost:3000
```

你会看到一个漂亮的 Web 界面，展示所有可用的 API 端点。

## 📚 API 文档

### 基础信息

- **Base URL**: `http://localhost:3000`
- **数据格式**: JSON
- **字符编码**: UTF-8

### 端点列表

#### 1️⃣ 获取所有用户

```http
GET /api/users
```

**响应示例：**
```json
{
  "success": true,
  "count": 3,
  "data": [
    {
      "id": 1,
      "name": "张三",
      "email": "zhangsan@example.com",
      "age": 25,
      "createdAt": "2024-03-24T10:30:00.000Z"
    },
    {
      "id": 2,
      "name": "李四",
      "email": "lisi@example.com",
      "age": 30,
      "createdAt": "2024-03-24T10:30:00.000Z"
    }
  ]
}
```

---

#### 2️⃣ 获取单个用户

```http
GET /api/users/:id
```

**参数：**
- `id` (必填) - 用户 ID

**响应示例：**
```json
{
  "success": true,
  "data": {
    "id": 1,
    "name": "张三",
    "email": "zhangsan@example.com",
    "age": 25,
    "createdAt": "2024-03-24T10:30:00.000Z"
  }
}
```

**错误响应：**
```json
{
  "success": false,
  "message": "未找到 ID 为 999 的用户"
}
```

---

#### 3️⃣ 创建新用户

```http
POST /api/users
Content-Type: application/json
```

**请求体：**
```json
{
  "name": "王五",
  "email": "wangwu@example.com",
  "age": 27
}
```

**字段说明：**
- `name` (必填) - 用户姓名
- `email` (必填) - 用户邮箱（必须唯一）
- `age` (可选) - 用户年龄，默认 0

**响应示例：**
```json
{
  "success": true,
  "message": "用户创建成功",
  "data": {
    "id": 4,
    "name": "王五",
    "email": "wangwu@example.com",
    "age": 27,
    "createdAt": "2024-03-24T12:00:00.000Z"
  }
}
```

---

### 使用 curl 命令

#### 1. 获取所有用户
```bash
curl http://localhost:3000/api/users
```

#### 2. 获取单个用户
```bash
curl http://localhost:3000/api/users/1
```

#### 3. 创建新用户
```bash
curl -X POST http://localhost:3000/api/users \
  -H "Content-Type: application/json" \
  -d '{
    "name": "赵六",
    "email": "zhaoliu@example.com",
    "age": 29
  }'
```

#### 4. 更新用户
```bash
curl -X PUT http://localhost:3000/api/users/1 \
  -H "Content-Type: application/json" \
  -d '{
    "name": "张三（已更新）",
    "age": 26
  }'
```

#### 5. 删除用户
```bash
curl -X DELETE http://localhost:3000/api/users/1
```

#### 6. 获取统计信息
```bash
curl http://localhost:3000/api/stats
```

#### 7. 下载 CSV
```bash
curl http://localhost:3000/api/users/export/csv -o users.csv
```

### 使用 JavaScript (Fetch API)

```javascript
// 获取所有用户
fetch('http://localhost:3000/api/users')
  .then(res => res.json())
  .then(data => console.log(data));

// 创建新用户
fetch('http://localhost:3000/api/users', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    name: '孙七',
    email: 'sunqi@example.com',
    age: 24
  })
})
  .then(res => res.json())
  .then(data => console.log(data));

// 更新用户
fetch('http://localhost:3000/api/users/2', {
  method: 'PUT',
  headers: {
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    name: '李四（已更新）',
    age: 31
  })
})
  .then(res => res.json())
  .then(data => console.log(data));

// 删除用户
fetch('http://localhost:3000/api/users/3', {
  method: 'DELETE'
})
  .then(res => res.json())
  .then(data => console.log(data));
```

## 💡 常见问题

### Q: 为什么不使用 Express？
A: 本项目旨在学习 Node.js 原生功能，理解底层原理。实际项目中使用 Express 会更高效。

### Q: 数据存储在文件中安全吗？
A: 这只是学习项目。生产环境应该使用专业数据库（如 PostgreSQL、MongoDB）。

### Q: 如何处理并发请求？
A: Node.js 是单线程异步模型，但可以处理大量并发。对于 CPU 密集型任务，考虑使用 Worker Threads。

### Q: CORS 是什么？
A: 跨域资源共享（CORS）允许前端从不同域名访问 API。我们在响应头中设置了 `Access-Control-Allow-Origin: *`。
