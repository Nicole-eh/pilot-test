# 高级功能：JWT 身份验证 + 角色权限控制

## 概述

本模块为项目新增了完整的 JWT（JSON Web Token）认证系统，包括：

- **用户注册**：密码使用 bcrypt 加密存储
- **用户登录**：签发 Access Token + Refresh Token（双 Token 机制）
- **Token 刷新**：用 Refresh Token 换取新的 Access Token
- **中间件鉴权**：验证请求中的 Bearer Token
- **角色权限控制**：区分 admin（管理员）和 user（普通用户）
- **退出登录**：撤销 Refresh Token

## API 端点

### 公开接口（无需 Token）

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | `/api/auth/register` | 用户注册 |
| POST | `/api/auth/login` | 用户登录 |
| POST | `/api/auth/refresh` | 刷新 Token |
| POST | `/api/auth/logout` | 退出登录 |

### 受保护接口（需要 Bearer Token）

| 方法 | 路径 | 权限 | 说明 |
|------|------|------|------|
| GET | `/api/auth/profile` | user/admin | 获取当前用户信息 |
| GET | `/api/auth/accounts` | admin | 查看所有账号 |
| DELETE | `/api/auth/accounts/:id` | admin | 删除指定账号 |

## 使用示例

### 1. 注册

```bash
curl -X POST http://localhost:3000/api/auth/register \
  -H "Content-Type: application/json" \
  -d '{"username":"zhangsan","password":"123456"}'
```

注册管理员：
```bash
curl -X POST http://localhost:3000/api/auth/register \
  -H "Content-Type: application/json" \
  -d '{"username":"admin","password":"admin123","role":"admin"}'
```

### 2. 登录

```bash
curl -X POST http://localhost:3000/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{"username":"zhangsan","password":"123456"}'
```

响应示例：
```json
{
  "success": true,
  "message": "登录成功",
  "data": {
    "user": { "id": 1, "username": "zhangsan", "role": "user" },
    "accessToken": "eyJhbGciOiJIUzI1NiIs...",
    "refreshToken": "eyJhbGciOiJIUzI1NiIs...",
    "expiresIn": "15m"
  }
}
```

### 3. 访问受保护接口

```bash
curl http://localhost:3000/api/auth/profile \
  -H "Authorization: Bearer eyJhbGciOiJIUzI1NiIs..."
```

### 4. 刷新 Token

当 Access Token 过期时，用 Refresh Token 换取新 Token：

```bash
curl -X POST http://localhost:3000/api/auth/refresh \
  -H "Content-Type: application/json" \
  -d '{"refreshToken":"eyJhbGciOiJIUzI1NiIs..."}'
```

### 5. 退出登录

```bash
curl -X POST http://localhost:3000/api/auth/logout \
  -H "Content-Type: application/json" \
  -d '{"refreshToken":"eyJhbGciOiJIUzI1NiIs..."}'
```

## 核心知识点

### 1. 双 Token 机制

```
Access Token（15分钟）  ── 用于 API 鉴权，短期有效，减少泄露风险
Refresh Token（7天）    ── 用于刷新 Access Token，长期有效，可单独撤销
```

**为什么需要两个 Token？**
- Access Token 短期有效，即使泄露影响有限
- Refresh Token 可被撤销，实现安全的退出登录
- 减少频繁登录，提升用户体验

### 2. 密码安全

```
明文密码 → bcrypt(password, salt_rounds=10) → 哈希值
```

- 从不存储明文密码
- bcrypt 内置盐值（salt），即使相同密码也产生不同哈希
- 计算开销大（10轮），暴力破解成本高

### 3. 认证流程

```
客户端                            服务器
  │                                │
  │  POST /auth/register           │
  │  {username, password}          │
  │──────────────────────────────→│  bcrypt(password) → 存储
  │                                │
  │  POST /auth/login              │
  │  {username, password}          │
  │──────────────────────────────→│  验证密码
  │  {accessToken, refreshToken}   │  签发双 Token
  │←──────────────────────────────│
  │                                │
  │  GET /auth/profile             │
  │  Authorization: Bearer <AT>    │
  │──────────────────────────────→│  jwt.verify(AT)
  │  {user profile}                │  解析用户信息
  │←──────────────────────────────│
  │                                │
  │  POST /auth/refresh            │  Access Token 过期后
  │  {refreshToken}                │
  │──────────────────────────────→│  验证 RT，签发新 AT+RT
  │  {newAccessToken, newRT}       │  撤销旧 RT
  │←──────────────────────────────│
```

### 4. 角色权限（RBAC）

```
admin  ──  可访问所有接口，可管理用户账号
user   ──  只能访问自己的信息
```

中间件链：`请求 → authenticate(验证Token) → authorize(检查角色) → 业务逻辑`

## Token 结构解析

一个 JWT Token 由三部分组成（用 `.` 分隔）：

```
Header.Payload.Signature

Header:    {"alg":"HS256","typ":"JWT"}
Payload:   {"id":1,"username":"admin","role":"admin","type":"access","iat":...,"exp":...}
Signature: HMACSHA256(base64(header)+"."+base64(payload), secret)
```

可以在 https://jwt.io 粘贴 Token 查看内容。
